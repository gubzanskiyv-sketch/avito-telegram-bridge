# Avito → Telegram

Сервис принимает webhook-события Messenger API Avito от нескольких профилей и публикует новые входящие сообщения в один Telegram-канал.

## Что уже реализовано

- три и более профиля Avito;
- подпись профиля в каждом уведомлении;
- защита webhook секретом в URL;
- фильтрация собственных исходящих сообщений;
- защита от повторной доставки одного события;
- безопасное экранирование текста для Telegram;
- отдельная очередь отправки, чтобы Avito быстро получал ответ `200 OK`;
- проверка состояния по адресу `/health`.

## 1. Подготовьте Telegram

1. Откройте `@BotFather`, выполните `/newbot` и сохраните токен.
2. Добавьте бота в нужный канал администратором с правом публикации.
3. Для публичного канала используйте `@имя_канала` как `TELEGRAM_CHAT_ID`.
4. Для частного канала перешлите сообщение из канала боту `@RawDataBot` и возьмите числовой `chat.id` вида `-100...`.

## 2. Заполните настройки

Скопируйте `.env.example` в `.env` и замените тестовые значения. Файл `.env` уже исключён из Git.

Для каждого профиля откройте в Avito раздел «Профиль → Интеграции и API» и возьмите `client_id` и `client_secret`. Не присылайте `client_secret` в переписку и не сохраняйте его в публичном репозитории.

Секрет webhook можно сгенерировать так:

```powershell
[guid]::NewGuid().ToString('N') + [guid]::NewGuid().ToString('N')
```

## 3. Разместите сервис

Нужен постоянно доступный HTTPS-адрес. Подойдут VPS с обратным прокси, Render, Railway, Fly.io или другой сервис, который запускает Docker-контейнер и выдаёт HTTPS-домен.

Приложение слушает порт из `PORT` (по умолчанию `8080`). После запуска откройте:

```text
https://ваш-домен/health
```

Ожидаемый ответ: `{"ok": true, ...}`.

## 4. Зарегистрируйте webhook для трёх профилей

Сначала безопасная проверка без запросов в Avito:

```powershell
python register_avito_webhooks.py --dry-run
```

Затем регистрация:

```powershell
python register_avito_webhooks.py
```

Скрипт получает отдельный токен для каждого профиля и подписывает все профили на адрес из `PUBLIC_WEBHOOK_BASE_URL`.

## 5. Проверьте

Напишите тестовое сообщение любому объявлению с другого аккаунта Avito. В канале должно появиться уведомление с названием профиля, ID отправителя, ID чата и текстом.

## Локальный тест

```powershell
python -m unittest -v
```

Локальный компьютер без публичного HTTPS-туннеля не сможет принимать webhook от Avito. Для постоянной эксплуатации лучше использовать сервер, а не временный туннель.

## Ограничение текущей версии

Это односторонний канал уведомлений. Для ответа клиенту из Telegram лучше использовать закрытую группу с ботом и кнопкой «Ответить»: публичный Telegram-канал для двусторонней работы неудобен и повышает риск случайно раскрыть переписку.
